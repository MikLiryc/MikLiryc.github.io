    GoMap GoTile 렌더 최적화

    발단

     현재 프로젝트에서, Lobby 씬에서 GoMap을 활용하여 현재 위치 중심의 지도를 표시한다. 지도는 GoTile 오브젝트로 이루어진 타일들로 이루어져 있으며, 각 타일에는 해당 위치의 도로, 강, 건물들에 해당하는 오브젝트가 생성된다. 화면에 렌더되는 타일의 개수가 많아지면 많아질 수록, 해당 타일 위의 오브젝트의 개수가 많아지고, 이에 따라 게임의 퍼포먼스가 떨어지는 현상이 발생하였다.

    현상 파악

     봉은사역 부근 패스트파이브 건물 기준으로 측정한 결과이다. (유니티 에디터)

     한 자리에서 화면을 360도 돌리면서 찾은 (그나마) 최소, 최대치의 Statistic 결과이다. 화면을 돌리기만 했을 뿐인데, Batches count 가 6000개 가량 차이가 나는 모습을 볼 수 있다. 이는 화면에 렌더되는 타일 중 도로, 강, 빌딩 등 오브젝트가 많은 타일이 있을 경우, 눈에띄게 큰 차이가 발생하며, 실제 게임 플레이 중 충분히 의미있는 영향을 주고 있다는 것을 보여준다.

    진행

     유니티의 Batches (draw call)은 렌더되는 오브젝트의 mesh * material 개수로 결정된다... 고 한다. 따라서, 동일한 material을 사용하는 오브젝트들 (도로는 도로끼리, 강은 강끼리, 빌딩은 빌딩끼리)의 mesh를 하나로 합치고, 하나의 material을 사용하여 렌더하도록 조정한다면, 오브젝트의 숫자만큼 늘어나던 batches를 하나 혹은 두 개로 줄일 수 있을 것으로 판단하였다.

    public class CombineMesh : MonoBehaviour
    {
    private Dictionary<string, CombineMeshData> _combineMeshDictionary = new();

    private void Start()
    {
    MeshFilter[] meshFilters = GetComponentsInChildren<MeshFilter>();
    MeshRenderer[] meshRenderers = GetComponentsInChildren<MeshRenderer>();

    // Material 별로 Dictionary에 정리
    ClassifyRenderers(meshFilters, meshRenderers);

    // 하나씩 꺼내서 Mesh combine 및 오브젝트 생성
    foreach (var combineMeshData in _combineMeshDictionary)
    {
    var go = new GameObject(combineMeshData.Value._sharedMaterial.name);
    go.transform.SetParent(transform);
    MeshFilter mf = go.AddComponent<MeshFilter>();
    MeshRenderer mr = go.AddComponent<MeshRenderer>();
    mr.sharedMaterial = combineMeshData.Value._sharedMaterial;
    mf.mesh = new Mesh();
    mf.mesh.indexFormat = UnityEngine.Rendering.IndexFormat.UInt32;
    mf.mesh.CombineMeshes(combineMeshData.Value._combineInstanceList.ToArray());
    }

    gameObject.RemoveComponentIfExists<MeshFilter>();
    gameObject.RemoveComponentIfExists<MeshRenderer>();
    gameObject.RemoveComponentIfExists<MeshCollider>();

    gameObject.SetActive(true);
    }

    // Material 별로 dictionary에 정리해놓기 위한 함수
    private void ClassifyRenderers(MeshFilter[] meshFilters, MeshRenderer[] meshRenderers)
    {
    for (int i = 0; i < meshRenderers.Length; i++)
    {
    Material mat = meshRenderers[i].sharedMaterial;
    if (!_combineMeshDictionary.TryGetValue(mat.name, out var meshData))
    {
    meshData = new(mat);
    _combineMeshDictionary.Add(mat.name, meshData);
    }

    CombineInstance instance = new();
    instance.mesh = meshFilters[i].sharedMesh;
    instance.transform = meshFilters[i].transform.localToWorldMatrix;
    meshData._combineInstanceList.Add(instance);
    meshFilters[i].gameObject.SetActive(false);
    }
    }
    }

    public class CombineMeshData
    {
    // Mesh 통합 후 사용할 Material
    public Material _sharedMaterial;
    public List<CombineInstance> _combineInstanceList = new();

    public CombineMeshData(Material material)
    {
    _sharedMaterial = material;
    }
    }

     Mesh 통합 이후, 기존 화면 대비 큰 차이가 있음을 확인 할 수 있다. 

     타일위의 Static / Dynamic 오브젝트의 갯수에 따라 조금씩 차이가 발생하기는 하지만, 전반적으로 Batches count 180~220 사이에서 유지되는 것을 확인할 수 있었고, FPS도 눈에 띄게 향상되었다.

     Dev 빌드를 제작하여 실기 테스트를 진행해본 결과, 기존 빌드에서는 가만히 있을 때는 30 프레임을 유지하다, 화면을 회전할 때마다 2~5 프레임정도 프레임 드랍이 발생하여 28~25 FPS를 보여주는 반면, Mesh 통합 적용 빌드에서는 화면을 회전하더라도 30프레임을 안정적으로 유지하는 모습을 보여주었다.
